Trilha de Desenvolvimento

Siga estas 3 fases essenciais para entregar um projeto funcional e conquistar seu badge

Fase 1
Planejamento

Planejamento e Documentação das Entregas

Objetivo: Transformar a ideia em um plano de ação claro e dividido em partes pequenas e realizáveis.
  • Escrever uma breve descrição do projeto (o que ele faz e para quem é)
  • Criar uma lista de funcionalidades essenciais (as primeiras entregas)
  • Montar um cronograma simples (ex: Trello, Notion ou planilha)
  • Entregar um documento explicando o escopo e as metas da primeira versão
Dica: pense como um cliente ou usuário final — o que é o mínimo necessário para o projeto ter valor?

Como Planejar um Projeto de Software do Zero Documentação de Projetos para Iniciantes Criando MVP: O que é e como fazer
Fase 2
Banco de Dados

Estruturação e Documentação do Banco de Dados

Objetivo: Definir o que realmente precisa ser salvo e como esses dados se relacionam.
  • Identificar os dados mais importantes (ex: usuários, tarefas, produtos)
  • Criar o modelo lógico do banco (tabelas, campos e relações)
  • Documentar decisões sobre tipos de dados e chaves primárias/estrangeiras
  • Criar o diagrama do banco (Draw.io, DB Diagram ou Lucidchart)
Dica: menos é mais — foque no essencial e deixe espaço para evoluir depois.
📚 Recursos Recomendados:
Modelagem de Banco de Dados para Iniciantes Como Criar Diagramas de Banco de Dados Normalização de Dados: Passo a Passo
Fase 3
Validação

Testar, Validar e Demonstrar Funcionando

Objetivo: Garantir que tudo que foi planejado realmente funciona e pode ser apresentado.
  • Testar as funcionalidades principais manualmente
  • Corrigir erros e ajustar o que não estiver funcionando como esperado
  • Criar um vídeo curto ou apresentação mostrando o projeto em ação
  • Escrever uma breve conclusão explicando o que foi aprendido
Dica: mesmo que algo não fique perfeito, o mais importante é mostrar evolução, aprendizado e domínio do processo.
